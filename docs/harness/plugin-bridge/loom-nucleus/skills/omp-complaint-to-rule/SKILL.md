# OMP Complaint To Rule

Use when the user voices a repeated complaint or correction that should become a durable, reviewable rule.

## Process

1. Restate the complaint as a single specific behavior to change.
2. Draft a concise rule with a clear trigger and the required behavior.
3. Present the draft for review; do not install it automatically.

## Rules

- Do not write or enable generated rules automatically; drafting is reviewable output only.
- Do not claim to control OMP runtime state.
- Keep each rule narrow, testable, and attributable to the originating complaint.
